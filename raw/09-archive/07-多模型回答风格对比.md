# 实验七：多模型回答风格对比

> [[Ollama LLM 实验系列索引]] | 同一 Prompt → 不同模型 → 看风格差异

## 实验方法

对同一观点判断类 Prompt（temperature=0.0），送给 4 个模型，比较回答风格、长度、判断取舍的差异。

**Prompt**：

> 阅读以下观点，请给出你的判断和分析：
> "远程办公（remote work）会降低团队效率，应该全面取消。"
> 请从正反两方面分析，然后给出你的最终判断。

**候选模型**：

| # | 模型 | 平台 | 类型 |
|:--|:-----|:-----|:-----|
| 1 | llama3.2:3b | Ollama（本地） | 3B 通用 |
| 2 | qwen3.5:0.8b | Ollama（本地） | 0.8B 通用 |
| 3 | DeepSeek V4 Pro | 云端 API | 云端 MoE |
| 4 | 豆包 Seed Character | 云端 API | 云端对话 |

## 实验结果（4 模型完整对比）

| 指标 | llama3.2:3b | qwen3.5:0.8b | 差异 |
|:----|:-----------:|:------------:|:----:|
| 输入 Token | 60 | 36 | qwen 分词效率高 1.7× |
| 输出 Token | **301** | **1936** | qwen 长 6.4× |
| 中文字数 | 475 | 1,404 | qwen 长 3.0× |
| 延迟 | **40.7s** | 121.5s | llama 快 3× |
| 生成速度 | 7.4 tok/s | 15.9 tok/s | qwen 快 2.1×（但输出多所以总慢） |

### 回答风格对比

**llama3.2:3b**（301 tokens, 475 字）— 简洁、要点式、中英混用：

```
正面分析：
1. 工作环境的变化：远程办公可以让员工在家中工作，从而减少了
   交通时间和通勤的压力...
2. flexible 工作模式：远程办公允许员工根据自己的需要选择工作
   方式...

反面分析：
1. 沟通障碍：远程办公可能影响团队成员之间的沟通和协作...
2. 管理难度：远程办公可能给管理层带来挑战...
```

**qwen3.5:0.8b**（1,936 tokens, 1,404 字）— 长篇大论、学术风、话痨：

```
关于"远程办公会降低团队效率，应该全面取消"这一观点，这是一个在
企业管理和人力资源领域非常常见且极具争议的话题。虽然该说法在某些
特定情境下可能成立（如强调沟通成本），但从正反两方面分析来看，其
核心逻辑存在明显的片面性。不能简单地认为远程办公必然降低效率并应
被完全取消，这取决于具体的组织目标、企业文化以及工作场景的复杂性。

以下是对这一观点从正面和反面两个维度的详细分析与判断...
```

### 风格差异总结

| 维度 | llama3.2:3b | qwen3.5:0.8b |
|:----|:-----------|:------------|
| **长度** | 短小精悍（475 字） | 长篇大论（1,404 字，3×） |
| **结构** | 要点列表 + 短句 | 段落论述 + 分析框架 |
| **语言** | 中英混用（flexible/office） | 纯中文书面语 |
| **立场** | 中立，正反并列表述 | 先批评原观点片面性，再分析 |
| **生成速度** | 慢（7.4 tok/s）但输出少 | 快（15.9 tok/s）但输出极多 |
| **总耗时** | **40.7s** | **121.5s**（慢 3×） |

## 分析

### 1. "小模型快"是陷阱

qwen3.5:0.8b 的原始生成速度（tok/s）是 llama3.2:3b 的 **2.1 倍**，但因为输出量多了 6.4 倍，总耗时反而慢了 3 倍。

> 测试简单 Prompt 时，"输出多少字"比"每秒生成多少字"更影响总耗时。

### 2. Fine-tuning 决定"话痨程度"

两个模型对同一 prompt 的回复长度差 **3 倍**（字数） / **6.4 倍**（tokens）。qwen3.5 的 fine-tuning 倾向于"先分析再下结论"的长篇格式，即使 temperature=0 也会输出完整的分析框架。这是训练数据的风格烙印，不是推理需求。

### 3. Tokenizer 差异持续存在

"远程办公"这类中文短语，qwen3.5 的 tokenizer（36 tokens）比 llama3.2（60 tokens）高效 **1.7 倍**，与之前实验六的发现一致。

### 4. llama3.2 中英混用的现象

llama3.2 在中文回复中混入英文词（flexible/office/remote work），而 qwen3.5 是纯中文。这可能与 **训练数据的中英文比例** 有关——llama3.2 的预训练数据英文占主导，中文是微调阶段加入的，模型在某些概念上"更习惯"用英文表达。

## 附：实验代码

```python
# 多模型对比脚本 — 核心部分
import os, sys, time, json, urllib.request
sys.stdout.reconfigure(encoding="utf-8", errors="replace")

PROMPT = "阅读以下观点并给出你的判断：\"远程办公会降低团队效率，\
应该全面取消。\" 请从正反两方面分析，然后给出你的最终判断。"
TEMPERATURE = 0.0

# === Ollama 本地 ===
def call_ollama(model):
    payload = {
        "model": model,
        "messages": [{"role": "user", "content": PROMPT}],
        "stream": False,
        "options": {"temperature": TEMPERATURE},
        "keep_alive": "10m",
    }
    req = urllib.request.Request("http://localhost:11434/api/chat",
        data=json.dumps(payload).encode("utf-8"),
        headers={"Content-Type": "application/json"})
    with urllib.request.urlopen(req, timeout=180) as resp:
        return json.loads(resp.read().decode("utf-8"))

# === DeepSeek (云端) ===
# 需要设置环境变量 DEEPSEEK_API_KEY
def call_deepseek():
    from openai import OpenAI
    client = OpenAI(base_url="https://api.deepseek.com",
                    api_key=os.environ["DEEPSEEK_API_KEY"])
    resp = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": PROMPT}],
        temperature=TEMPERATURE)
    return resp

# === 豆包 (云端) ===
# 需要设置环境变量 DOUBAO_API_KEY
def call_doubao():
    from openai import OpenAI
    client = OpenAI(
        base_url="https://ark.cn-beijing.volces.com/api/v3",
        api_key=os.environ["DOUBAO_API_KEY"])
    resp = client.chat.completions.create(
        model="doubao-seed-character-260628",
        messages=[{"role": "user", "content": PROMPT}],
        temperature=TEMPERATURE)
    return resp

# === 运行全部模型 ===
results = []
for name, fn in [
    ("llama3.2:3b", lambda: call_ollama("llama3.2:3b")),
    ("qwen3.5:0.8b", lambda: call_ollama("qwen3.5:0.8b")),
    ("DeepSeek V4 Flash", call_deepseek),
    ("豆包 Seed Character", call_doubao),
]:
    print(f"[{name}] ...", end=" ", flush=True)
    try:
        t0 = time.time()
        r = fn()
        lat = time.time() - t0
        text = r.choices[0].message.content \
            if hasattr(r, 'choices') else r["message"]["content"]
        it = r.usage.prompt_tokens \
            if hasattr(r, 'usage') else r["prompt_eval_count"]
        ot = r.usage.completion_tokens \
            if hasattr(r, 'usage') else r["eval_count"]
        results.append((name, lat, it, ot, len(text), text))
        print(f"OK {lat:.1f}s  in={it} out={ot} ({len(text)}字)")
    except Exception as e:
        print(f"FAIL: {e}")

# === 汇总输出 ===
print(f"\n{'='*60}")
print(f"{'模型':<22} {'延迟':>6} {'输入T':>5} {'输出T':>5} {'字数':>5}")
print("-"*50)
for r in results:
    print(f"{r[0]:<22} {r[1]:>5.1f}s {r[2]:>4d} {r[3]:>4d} {r[4]:>4d}")

if results:
    chars = [r[4] for r in results]
    sn = results[chars.index(min(chars))]
    lx = results[chars.index(max(chars))]
    print(f"\n📏 最短: {sn[0]} ({sn[4]}字)")
    print(f"📏 最长: {lx[0]} ({lx[4]}字, {lx[4]/sn[4]:.1f}倍)")
    ft = min(results, key=lambda r: r[1])
    sl = max(results, key=lambda r: r[1])
    print(f"⚡ 最快: {ft[0]} ({ft[1]:.2f}s)")
    print(f"🐢 最慢: {sl[0]} ({sl[1]:.2f}s)")

# 输出每个模型的完整回复
for r in results:
    print(f"\n\n{'='*60}")
    print(f"  【{r[0]}】— {r[4]}字 / {r[3]} tokens")
    print(f"{'='*60}")
    print(r[5])
```
