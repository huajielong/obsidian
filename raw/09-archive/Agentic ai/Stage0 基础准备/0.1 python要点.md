# 0.1 python要点

我们来把这三种模型都放到**真实业务场景**里讲清楚，每种给你一段可跑的代码 + 逐行解释。

---

## 一、async/await：单线程并发，适合 IO 密集（比如：批量请求 HTTP 接口）

### 业务场景

你要**同时请求很多第三方 HTTP 接口**（比如拉取多个服务的状态、行情、天气、推荐结果），每个请求都在“等网络”，CPU 其实很闲——典型 IO 密集。

### 示例代码（asyncio + aiohttp）

```Python
import asyncio
import aiohttp

URLS = [
    "https://httpbin.org/delay/2",
    "https://httpbin.org/delay/2",
    "https://httpbin.org/delay/2",
]

async def fetch(session, url):
    print(f"start fetch: {url}")
    async with session.get(url) as resp:
        data = await resp.text()
        print(f"done fetch: {url}, status={resp.status}")
        return data

async def main():
    async with aiohttp.ClientSession() as session:
        tasks = [fetch(session, url) for url in URLS]
        results = await asyncio.gather(*tasks)
        print(f"got {len(results)} responses")

if __name__ == "__main__":
    asyncio.run(main())
```

### 逐行解释

- `import asyncio`    
**导入事件循环框架**，负责调度协程。
- `import aiohttp`    
**异步 HTTP 客户端库**，支持 `await session.get(...)`。
- `URLS = [...]`    
**要并发请求的 URL 列表**，模拟多个接口。
- `async def fetch(session, url):`    
 定义一个**异步函数**，负责请求单个 URL。
- `print(f"start fetch: {url}")`    
 打印开始时间，方便观察并发效果。
- `async with session.get(url) as resp:`    
 使用 `async with` 打开异步 HTTP 请求，**不会阻塞线程**，只阻塞当前协程。
- `data = await resp.text()`    
**等待网络数据返回**，期间事件循环可以去跑其他协程。
- `print(f"done fetch: {url}, status={resp.status}")`    
 请求完成后打印状态码。
- `return data`    
 返回响应内容。
- `async def main():`    
 主协程，负责创建 session 和任务。
- `async with aiohttp.ClientSession() as session:`    
 创建一个 HTTP 会话，复用连接。
- `tasks = [fetch(session, url) for url in URLS]`    
**只是创建协程对象**，还没真正执行。
- `results = await asyncio.gather(*tasks)`    
 把多个协程打包并发执行，**单线程高并发 IO** 的关键。
- `print(f"got {len(results)} responses")`    
 打印总共拿到多少个响应。
- `if name == "main":`    
 只在脚本直接运行时执行。
- `asyncio.run(main())`    
 创建事件循环并运行 `main` 协程。

## 二、多线程：适合“混合任务”或**阻塞 IO 库**（比如：同时读多个大文件 + 做一点轻量计算）

### 业务场景

你有一堆**大文件要读取**（磁盘 IO），同时每个文件读完后要做一点**轻量级解析/统计**。  
 这些库是**同步阻塞的**，又不想改成 asyncio，就用多线程把 IO 并发起来。

### 示例代码（ThreadPoolExecutor）

```Python
import time
from concurrent.futures import ThreadPoolExecutor, as_completed
from pathlib import Path

FILES = ["file1.log", "file2.log", "file3.log"]

def process_file(path: str):
    print(f"[{path}] start read")
    # 模拟磁盘 IO
    time.sleep(2)
    # 模拟简单解析
    line_count = 1000  # 假设解析出 1000 行
    print(f"[{path}] done, lines={line_count}")
    return path, line_count

def main():
    with ThreadPoolExecutor(max_workers=3) as executor:
        futures = [executor.submit(process_file, f) for f in FILES]
        for future in as_completed(futures):
            path, count = future.result()
            print(f"result: {path} -> {count}")

if __name__ == "__main__":
    main()
```

### 逐行解释

- `import time`    
 用 `sleep` 模拟磁盘 IO。
- `from concurrent.futures import ThreadPoolExecutor, as_completed`    
**线程池工具**，方便管理多线程任务。
- `from pathlib import Path`    
 这里可以用来处理文件路径（示例中没深用）。
- `FILES = [...]`    
 要处理的文件列表。
- `def process_file(path: str):`    
 在线程中执行的**普通同步函数**，不是 async。
- `print(f"[{path}] start read")`    
 打印开始处理哪个文件。
- `time.sleep(2)`    
 模拟**阻塞的磁盘 IO**，线程在这里会阻塞，但其他线程可以继续跑。
- `line_count = 1000`    
 模拟解析结果，比如统计行数。
- `print(f"[{path}] done, lines={line_count}")`    
 打印处理完成。
- `return path, line_count`    
 返回结果，方便主线程汇总。
- `def main():`    
 主函数。
- `with ThreadPoolExecutor(max_workers=3) as executor:`    
 创建一个**最多 3 个工作线程**的线程池。
- `futures = [executor.submit(process_file, f) for f in FILES]`    
 把每个文件的处理任务提交给线程池，立即返回 `Future` 对象。
- `for future in as_completed(futures):`    
 按照任务完成的顺序迭代结果，而不是提交顺序。
- `path, count = future.result()`    
 阻塞等待某个任务完成并拿到返回值。
- `print(f"result: {path} -> {count}")`    
 打印每个文件的处理结果。
- `if name == "main":`    
 入口保护。
- `main()`    
 启动程序。

## 三、多进程：适合 CPU 密集（比如：批量图片处理 / 大量数值计算）

### 业务场景

你要对**大量图片做滤镜/特征提取**，或者做**大规模数值计算**（比如模拟、加密、机器学习预处理），CPU 打满才是目标——典型 CPU 密集，用多进程吃满多核。

### 示例代码（multiprocessing 计算大量数字的平方和）

```Python
from multiprocessing import Pool, cpu_count
import time

def heavy_compute(n: int) -> int:
    total = 0
    for i in range(10_000_000):
        total += (i * i) % n
    return total

def main():
    numbers = [101, 103, 107, 109]  # 模拟不同任务参数

    print(f"cpu_count = {cpu_count()}")

    start = time.time()
    with Pool(processes=4) as pool:
        results = pool.map(heavy_compute, numbers)
    end = time.time()

    print("results:", results)
    print("time used:", end - start)

if __name__ == "__main__":
    main()
```

### 逐行解释

- `from multiprocessing import Pool, cpu_count`    
 导入**进程池**和获取 CPU 核心数的函数。
- `import time`    
 用来测量耗时。
- `def heavy_compute(n: int) -> int:`    
 定义一个**CPU 密集型函数**，会做大量循环和运算。
- `total = 0`    
 初始化累加器。
- `for i in range(10_000_000):`    
 大循环，模拟重计算。
- `total += (i * i) % n`    
 做一些算术运算，确保 CPU 真在干活。
- `return total`    
 返回计算结果。
- `def main():`    
 主函数。
- `numbers = [101, 103, 107, 109]`    
 模拟 4 个不同的计算任务参数。
- `print(f"cpu_count = {cpu_count()}")`    
 打印当前机器的 CPU 核心数，方便对比。
- `start = time.time()`    
 记录开始时间。
- `with Pool(processes=4) as pool:`    
 创建一个**4 进程的进程池**，通常设为 CPU 核心数或略少。
- `results = pool.map(heavy_compute, numbers)`    
 把 `numbers` 中的每个参数分发到不同进程并行计算，返回结果列表。
- `end = time.time()`    
 记录结束时间。
- `print("results:", results)`    
 打印每个任务的计算结果。
- `print("time used:", end - start)`    
 打印总耗时，通常会明显比单进程快。
- `if name == "main":`    
 Windows 下多进程必须有这行，否则会无限递归创建子进程。
- `main()`    
 启动程序。

## 小结：三种模型 + 典型业务场景

- **async/await（单线程并发 IO）**    
**场景：** 高并发 HTTP 请求、爬虫、网关、聊天服务、长连接推送。  
**特点：** 事件循环调度协程，IO 等待时切任务。
- **多线程（混合任务 / 阻塞 IO 库）**    
**场景：** 同时读写多个文件、调用阻塞第三方 SDK、日志采集、同步数据库驱动。  
**特点：** 线程之间共享内存，适合 IO 多、CPU 适中。
- **多进程（CPU 密集）**    
**场景：** 图像处理、视频转码、数值计算、机器学习预处理、加密/压缩。  
**特点：** 每个进程独立 GIL，多核并行，适合纯计算。
