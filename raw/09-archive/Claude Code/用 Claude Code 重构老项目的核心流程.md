# 用 Claude Code 重构老项目的核心流程

可分为以下 5 步，结合视频中的实操经验整理：

1. 开启 Plan Mode，制定重构蓝图

工具操作：按两次 Shift+Tab 或输入/plan进入规划模式，让 Claude Code 读取项目代码、分析模块关系、文件依赖关系（只读不改）。

输出产物：生成plan.md，记录重构目标（如 “提升代码可读性”）、风险点（如 “核心认证逻辑”）、待办清单。

注意事项：AI 的理解有限，需手动补充业务逻辑说明，避免 “屎山平移”。



1. 建立安全网：补全测试用例

工具操作：让 Claude Code 自动生成单元测试（如针对 Vue 组件的mount测试），优先覆盖核心函数（如数据流转逻辑）。

目标指标：将测试覆盖率从低水平（如 10%）提升至 40%+，确保改动后能快速发现错误。

注意事项：测试需覆盖边界情况（如空值、异常），避免 AI 漏判。



1. 小步快跑：分阶段重构

操作原则：每次只改一个叶子节点逻辑单元（叶子函数、文件、模块），完成后立即测试→提交→再进行下一步。

注意事项：

先保证功能不崩，再优化代码风格。



1. 设置 Hard Rules，保护承重墙

工具操作：在plan.md中标记禁止改动的模块（如数据库迁移文件、认证逻辑），作为 AI 的 “硬规则”。

注意事项：老项目的 “承重墙”（如核心算法）即使代码丑也不能动，避免系统崩溃。



1. 翻车复盘：回滚与迭代

工具操作：

回滚：发现逻辑错误后，立即回滚到上一个正常版本。

复盘：与 Claude Code 分析失败原因（如边界条件漏判），将经验补充到plan.md。

注意事项：复杂重构可能需要 4-5 轮迭代，属于正常现象，需耐心优化。



关键经验总结

AI 角色：Claude Code 是 “执行者”，不是 “决策者”，需人工主导方向。

效率提升：视频案例显示，1 万行代码重构可缩短 50% 时间，但人工审查仍不可少。

核心原则：先稳后美，功能稳定优先于代码优雅
