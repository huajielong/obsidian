# 实验二：温度参数扫描与取值范围

> [[Ollama LLM 实验系列索引]] | 模型：llama3.2:3b

## 实验方法

对同一 Prompt（"写一句关于人工智能的话，不超过20个字。"），从 `temperature=0.0` 到 `1.5` 扫描 8 档温度，每档重复 5 次，观察输出 Token 数的变化和内容多样性。

## 实验结果

| 温度 | 输出 Token（5 次） | 多样性 | 输出样例（部分） |
|:---:|:-----------------:|:-----:|----------------|
| **0.0** | 30 / 30 / 30 / 30 / 30 | **1/5** | 人工智能是利用计算机和算法来模拟人类智能的技术... |
| 0.1 | 30 / 30 / 34 / 30 / 30 | 2/5 | 同上 / 人工智能是利用计算机和算法来模拟人类思维... |
| 0.3 | 25 / 33 / 28 / 28 / 30 | **5/5** | 人工智能是利用计算机和算法来模拟人类大脑的能力... |
| 0.5 | 27 / 27 / 36 / 28 / 32 | 5/5 | 人工智能是利用计算机和算法来模拟人类大脑思维... |
| 0.7 | 28 / 26 / 32 / 28 / 27 | 5/5 | 人工智能是指通过计算机和算法实现的智能行为... |
| 0.9 | 22 / 31 / 28 / 28 / 29 | 5/5 | 人工智能是利用计算机和数学技术来模拟人类intelligence... |

## 分析

### 温度取值范围

Ollama/OpenAI API 规范范围为 **0.0 ~ 2.0**：
- `0.0` — 贪婪解码，完全确定
- `1.0` — 默认值，标准采样
- `2.0` — 最高随机性
- 超出 2.0 会被截断；负值极少使用

### 温度效应

**温度 = 0（贪婪解码）**：
- 输出完全确定，每次 Token 数和内容一致
- 模型每次都选择概率最高的 token

**温度 > 0（采样解码）**：
- 即使温度仅 0.1，5 次输出即开始出现不同（多样性 2/5）
- `0.5~0.7` 在多样性和合理性之间取得平衡 —— 大多数应用的推荐范围
- `1.5` 时输出变短（均仅 27.8 Token），且英文单词 `intelligence`/`smart` 直接出现——模型在高温度下更容易"跳出"中文语境

**长度波动**：温度越高，输出长度的极差越大——`0.9` 时极差 15（22~37），说明高温度下模型对输出长度的控制力下降。

## 附：实验代码

```python
# 温度扫描 — 取自 ollama_experiment_v3.py
prompt = "写一句关于人工智能的话，不超过20个字。"
temps = [0.0, 0.1, 0.3, 0.5, 0.7, 0.9, 1.0, 1.5]

for temp in temps:
    outs, txts = [], []
    for _ in range(5):
        r = call_ollama(prompt, temperature=temp)
        outs.append(r["eval_count"])
        txts.append(r["message"]["content"])
    avg = sum(outs) / len(outs)
    uc = len(set(txts))
    print(f"温度 {temp:>3.1f} | 输出Token {avg:5.1f} "
          f"(min={min(outs)}, max={max(outs)}) | 多样性 {uc}/5 | "
          f"{txts[0][:35]}")
```

> `call_ollama` 实现见 [[Ollama LLM 实验系列索引#核心 API 函数（通用）]]
