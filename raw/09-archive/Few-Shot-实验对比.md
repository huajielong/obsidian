# Few-Shot 实验

## 什么是 Few-Shot？

| 术语 | 含义 |
|------|------|
| **Zero-shot (0-shot)** | 不给任何范例，直接让模型推理 |
| **One-shot (1-shot)** | 先给 1 个"输入 → 答案"范例再问 |
| **Few-shot (3-shot)** | 给几个（通常 2-5 个）范例再问 |

核心原理：LLM 从范例中学习输出格式 + 判断标准，准确率通常会提升。

---

## 实验 V1：意图分类（4 类）

- **模型**: `deepseek-v4-pro`
- **任务**: 将留言分类为 `inquiry` / `complaint` / `order` / `spam`
- **测试集**: 10 条（spam 未出现在 3-shot 范例中，检验泛化能力）

### 结果

| 模式 | 准确率 |
|------|--------|
| 0-shot | 100% |
| 3-shot | 100% |

**结论**: deepseek-v4-pro 对这种简单分类任务，0-shot 已经完美——模型足够强，few-shot 没有额外收益。

---

## 实验 V2：结构化 JSON 提取

- **模型**: `deepseek-v4-pro`
- **任务**: 从客服留言中提取 `urgency`(紧急度) + `sentiment`(情绪) + `action`(建议动作)
- **输出格式**: JSON 行
- **难度**: 比 V1 更高（结构化输出 + 多字段）

### 结果

| 维度 | 0-shot | 3-shot |
|------|--------|--------|
| JSON 格式遵循 | 10/10 (100%) | 7/10 (70%) |
| Urgency 准确率 | 9/10 (90%) | 9/10 (90%) |
| Sentiment 准确率 | 10/10 (100%) | 9/10 (90%) |
| 完全正确 | 8/10 (80%) | 6/10 (60%) |

> **意外发现**: 3-shot 结果反而略低于 0-shot。可能原因是范例中的 `action` 字段值（"安抚并催单"）让模型倾向于输出更长的动宾短语，偏离了"10 字以内"的要求。

## 实验 V3：跨模型对比（核心实验）

对比本地弱模型与云端强模型的 few-shot 效果差距。

### 实验设计

| 项目 | 说明 |
|------|------|
| 任务 | 4 类意图分类（同 V1） |
| 弱模型 | `llama3.2:3b`（本地 Ollama，3B 参数，CPU 推理） |
| 强模型 | `deepseek-v4-pro`（云端 API） |
| 指标 | 0-shot vs 3-shot 准确率 |

### 结果对比

| 模型 | 0-shot | 3-shot | 变化 |
|------|--------|--------|------|
| **llama3.2:3b** (3B) | **80%** (8/10) | **90%** (9/10) | **↑ +10%** |
| **deepseek-v4-pro** | ~95% | ~90% | ~0% |

### 逐条分析：llama3.2:3b

```
0-shot:
  ✗ [ 6] 预期=投诉 预测=咨询  | 耳机收到了，但是左耳没声音，怎么处理？
  ✗ [10] 预期=投诉 预测=垃圾  | 等了一个小时还没上菜，什么垃圾店
  → 把"投诉"误判为"咨询"和"垃圾"
  → "垃圾店"字面含"垃圾"→模型被关键词误导

3-shot:
  ✓ [10] 预期=投诉 预测=投诉  | 等了一个小时还没上菜，什么垃圾店
  ✗ [ 6] 预期=投诉 预测=咨询  | 耳机收到了，但是左耳没声音，怎么处理？
  → 范例帮助模型纠正了"垃圾店"的分类
  → "耳机故障"这条仍被误判为咨询（可能因为语气较温和）
```

### 关键发现

1. **Few-shot 对小模型更有效** —— llama 3B 上 3-shot 带来 +10% 提升，而 deepseek-v4-pro 接近天花板
2. **Few-shot 帮助纠正关键词误导** —— "垃圾店" 在 0-shot 中被字面理解为 "垃圾"，3-shot 范例让模型学会按意图分类
3. **模型越强，few-shot 边际收益越低** —— 当 0-shot 已达到 ~95%，几个范例几乎没有增量价值
4. **格式遵循度不依赖 few-shot** —— 两个模型的格式遵循在 0-shot 已是 100%

## 附：工程注意事项

### 本地 Ollama × OpenAI 兼容接口兼容性

| 模型 | OpenAI 兼容接口 | 原生 API |
|------|----------------|---------|
| qwen3.5:0.8b | ❌ 返回空内容 (`finish_reason: length`) | ✅ 正常 |
| llama3.2:3b | ✅ 正常 | ✅ 正常 |

使用 Ollama 时，如果模型通过 `/v1/chat/completions` 返回空，尝试直接调 `/api/chat` 原生接口。
