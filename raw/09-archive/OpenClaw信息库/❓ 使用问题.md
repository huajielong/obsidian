# OpenClaw 使用问题

> 更新时间：2026-03-19

---

## 🔧 启动报错

### ❌ Node.js版本问题

**症状**：启动失败，提示版本不兼容

**解决**：

```Bash
# 检查版本
node -v

# 要求 v18+ 或 v20+
# 建议使用 nvm 管理版本
nvm install 20
nvm use 20

```

---

### ❌ native依赖缺失

**症状**：提示缺少 native 模块

**解决**：

```Bash
npm rebuild
# 或重新安装
rm -rf node_modules
npm install

```

---

## 🔧 API配置问题

### ❌ 模型API无效

**症状**：提示API key无效

**解决**：

1. 检查API Key是否正确
2. 确认模型名称匹配
3. 查看余额是否充足

**配置示例**（阿里云百炼）：

```Plain Text
模型格式：alibaba-cloud/模型名
如：alibaba-cloud/qwen-plus

```

**来源**：[阿里云帮助中心](https://help.aliyun.com/zh/simple-application-server/use-cases/openclaw-faq)

---

### ❌ GPT/Claude无法连接

**症状**：网络超时或连接拒绝

**解决**：

1. 检查代理设置
2. 确认防火墙放行
3. 尝试更换模型

---

## 🔧 Skill加载失败

### ❌ 技能无法启用

**症状**：skill状态显示异常

**解决**：

1. 运行诊断：`openclaw doctor`
2. 检查目录结构
3. 查看错误日志

**来源**：[腾讯云](https://cloud.tencent.com/developer/article/2634710)

---

## 🔧 端口冲突

### ❌ 3000端口被占用

**症状**：Gateway启动失败

**解决**：

```Bash
# 查找占用进程
lsof -i :3000

# 杀掉进程或修改配置
# 编辑 config.yaml 修改端口

```

## 🔧 常见问题速查

| 问题 | 解决方案 |
|-|-|
| 安装后无法启动 | 检查Node.js版本 |
| API报错 | 检查密钥和网络 |
| Skill不生效 | 重启或重新安装 |
| 消息发不出去 | 检查channel配置 |
| 内存占用高 | 清理缓存或升级配置 |

## 🛠️ 诊断工具

**推荐**：使用内置诊断命令

```Bash
openclaw doctor

```

可自动检测：

- 配置缺失
- 端口冲突
- 认证失效
- 依赖问题
