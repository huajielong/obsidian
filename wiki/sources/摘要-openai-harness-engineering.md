# OpenAI Harness Engineering：驾驭工程

> **人类掌舵。智能体执行。**

OpenAI 官方于 2026 年 2 月 11 日发布的实验报告，记录了团队在 5 个月内从零构建一个内部 Beta 产品——**没有一行代码是人类手动编写的**。

## 核心数据

| 指标 | 数据 |
|------|------|
| 团队规模 | 3人 → 7人 |
| 时间周期 | 2025年8月 → 2026年1月（5个月） |
| 代码总量 | **约100万行** |
| PR 数量 | **约1500个** |
| 人均吞吐量 | **每人每天3.5个PR** |
| 效率提升 | 约 **10倍** 于传统手写代码 |
| 模型 | GPT-5（Codex CLI） |

## 关键原则

1. **不手动编写代码** — 从应用逻辑、测试、CI配置、文档到内部工具，全部由 Codex 生成
2. **人类掌舵，智能体执行** — 人类负责设计环境、明确意图、构建反馈回路
3. **智能体可读性优先** — 代码仓库本身是所有知识的记录系统，不在代码仓库外的知识对智能体而言"不存在"

## 核心架构模式

### 知识分层（渐进式披露）
- `AGENTS.md` — 仅约100行，作为内容目录/地图
- `docs/` 目录 — 结构化知识库，作为记录系统
- 设计文档、执行计划、产品规格、架构文档均被编目索引
- 专职 linter + "doc-gardening" 智能体维护文档新鲜度

### 三层自动质检（推理三明治）
1. **顶层** 高推理模型（GPT-5）— 规划与拆解任务
2. **中层** 低推理模型 — 低成本批量执行
3. **底层** 高推理模型 — 按需质检，仅对高风险变更深度审查

### 架构约束
- 每个业务域划分为固定层：Types → Config → Repo → Service → Runtime → UI
- 依赖方向严格验证，通过自定义 linter 和结构测试强制执行
- 横切关注点（认证、遥测、功能标志）通过单一显式接口 `Providers` 进入

### 可观测性赋能
- 日志（LogQL）、指标（PromQL）、追踪（TraceQL）对 Codex 完全开放
- Chrome DevTools 协议接入智能体运行时，支持 DOM 快照、截图、导航
- 每个 git worktree 启动独立应用实例，任务完成后彻底清理

### 垃圾回收机制
- "黄金原则"直接编码到仓库中
- 定期运行后台清理任务扫描偏差、修复技术债
- 持续以小额贷款偿还技术债，避免一次性痛苦解决

## 关键洞察

- **AGENTS.md 不是百科全书，而是内容目录** — 实践证明庞大指令文件会挤掉任务上下文、快速腐烂、难以验证
- **吞吐量改变合并理念** — 纠错成本低，等待成本高；减少阻塞合并门
- **枯燥的技术更受智能体青睐** — 可组合、API 稳定、在训练集中表现好
- **品味编码化** — 人类品味通过文档更新或直接编码为工具规则持续反馈到系统中
