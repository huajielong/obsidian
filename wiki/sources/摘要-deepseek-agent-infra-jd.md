# DeepSeek Agent Infra 研发工程师职位描述

> **来源**：DeepSeek 官方招聘信息（2026 年），由 Claude 获取并整理。
>
> **核心定位**：**DSec** 是为 Agent 量身定制的云平台，托管着成千上万个沙箱环境，有序推进着下一代模型的迭代。

---

## 团队使命

DSec 需要修改整个系统栈——从操作系统到虚拟机，再到各级网络和存储，一直上到应用层调度和管控面服务——来增加 Agent 运行环境的可靠性和效率。

> 你需要解决各种规模的系统问题——小到自旋锁，大到跨区域数据一致性——还要加上无监管全自动运行的 Agent 带来的更多安全挑战。以往对负载模式、数据隔离、资源用量、平台安全的种种假设都可能不复成立，需要另辟蹊径。

---

## 可选方向

| 方向 | 核心工作 | 关键技术 |
|------|---------|---------|
| **虚拟化环境** | 托管大规模 VM 集群，支持多样 OS 配置和硬件方案 | KVM/QEMU、Hypervisor、硬件虚拟化 |
| **容器化** | 加固隔离、缩小逃逸面、降低沙箱资源占用量 | Docker/containerd、Kubernetes、Seccomp/AppArmor |
| **临时存储** | 设计专供沙箱的块设备和共享卷方案 | Ceph、分布式文件系统、NVMe |
| **虚拟化网络** | 超大规模虚拟网络，Agent 在物理边界内互通信 | VPC、OVS、eBPF、RDMA |
| **云服务** | 混合云架构、可观测平台、日志系统、控制面 | Kubernetes、Prometheus、Tracing |
| **可观测性和高可用性** | 监控、节点自动上下线、应急响应 | SLO/SLI、告警体系、故障自愈 |

## 关键洞察

### 1. 你是 Agent 的"房东"

> 你来决定新时代 Agent 基础设施的形态，以及 Agent 们的生活环境。

这句 JD 中的话精准定义了 Agent Infra 的定位——它不是 Agent 本身，而是 **Agent 运行的家**。DSec 工程师决定：
- Agent 的沙箱有多大（资源配额）
- Agent 之间能否串门（网络策略）
- Agent 能带什么进来（数据注入/隔离）
- 环境安不安全（逃逸防护）

### 2. Sandbox 是 Harness Engineering 的物理实现层

在 [[Harness_Engineering]] 的 8 核心元件中，"Safety Layer"（#4）和"Sandboxed Execution"是抽象概念。**DSec 是这些概念的具体物理实现**：

| Harness 抽象元件 | DSec 物理实现 |
|-----------------|-------------|
| Safety Layer | 虚拟化/容器隔离、Seccomp、AppArmor |
| Tool Registry | 网络策略、数据沙箱、API 权限管控 |
| Retry/Recovery | 节点自动上下线、故障自愈、跨区域容灾 |
| Observatory | 可观测平台、日志系统、监控告警 |
| Context Manager | 临时存储设计、块设备/共享卷供给 |

### 3. "无监管全自动运行"带来的范式颠覆

> 以往对负载模式、数据隔离、资源用量、平台安全的种种假设都不复成立。

这揭示了 Agent 沙箱与传统云平台的根本差异：

| 维度 | 传统云平台 | Agent 沙箱（DSec） |
|------|----------|-------------------|
| **负载模式** | 可预测、稳态 | 突发、不可预测、Agent 行为随机性大 |
| **数据隔离** | 租户间隔离明确 | Agent 间需要精细控制（可协作可隔离） |
| **资源用量** | 固定配额 | 动态变化（Agent 可能突然扩缩） |
| **安全假设** | 受控用户操作 | 无监督 Agent 可能尝试任何操作 |
| **故障模式** | 已知模式 | Agent 行为引入新的故障类型 |
| **生命周期** | 长稳服务 | 沙箱短生命周期、频繁创建销毁 |

### 4. "全系统栈"覆盖

JD 描述了一条完整的系统栈能力路径：

```
自旋锁（微观）                    ← 内核/OS 层面
    ↓
操作系统/虚拟化                    ← 系统软件层面
    ↓
虚拟机/容器                       ← 虚拟化层面
    ↓
虚拟网络 / 存储                    ← 基础设施层面
    ↓
调度 / 控制面 / 可观测性          ← 平台层面
    ↓
跨区域数据一致性（宏观）            ← 分布式系统层面
```

一个人/团队需要理解从 CPU 微架构到跨区域分布式系统的 **全栈能力**。

### 5. 系统顶会的"学术信号"

加分项列出 OSDI/SOSP/EuroSys/NSDI/ATC——这五个是操作系统、存储、网络、虚拟化领域**最顶尖的系统会议**。这反映：
- DeepSeek 愿意招纯系统方向的 PhD（不要求 ML 背景）
- DSec 面对的工程难度达到了学术前沿水准
- 系统能力是 Agent 时代的核心基础设施竞争壁垒
