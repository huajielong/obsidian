# AI创作数据产品经理职位描述

## 来源概览

- **职位**: AI创作数据产品经理
- **公司信号**: DeepSeek
- **团队使命**: "探索AI创造力的上限，以人类最高审美标准，驱动模型创作能力持续进化"
- **核心关键词**: 审美标准、写作能力、文学创作、功能写作、数据质量

---

## 关键洞察

### 洞察 1：角色定位——三部曲的最后一环

本 JD 与 JD 11（通用数据产品经理）、JD 12（专业领域数据产品经理）构成**评测数据产品经理三部曲**：

```
JD 11：通用场景数据产品经理  →  办公/生活/搜索（最大用户群）
JD 12：专业领域数据产品经理  →  小语种/医学/法律（高知识密度）
本 JD：AI创作数据产品经理    →  文学/写作/审美（最抽象前沿）
                          ↓
共同方法论：评测体系 + 数据管线 + 理想态定义
不同点：领域知识不同，质量判断标准不同
```

三部曲覆盖了模型能力评测数据的完整光谱：

| 维度 | JD 11（通用） | JD 12（专业） | 本 JD（创作） |
|------|-------------|-------------|-------------|
| **领域** | 日常场景 | 专业知识 | **审美创作** |
| **核心能力** | 场景拆解 + 数据工程 | 专家判断 + 评测设计 | **审美判断 + 文学理解** |
| **质量定义者** | 用户需求 | 专业标准 | **人类审美最高标准** |
| **评测客观性** | 中（可用自动化） | 中高（三维度框架） | **低（最依赖主观判断）** |
| **与模型的关系** | 改进驱动者 | 质量校准者 | **天花板探索者** |

### 洞察 2："AI 的壁垒，不过是它尚未抵达的前沿"

> "写作与审美是人类认知中最抽象、最深邃的疆域——我们坚信这并非 AI 的壁垒，不过是它尚未抵达的前沿。"

这是十二份 JD 中**最具胆识的使命宣言**。它传达了几个重要信号：

1. **审美不是 AI 的禁区**——不同于"AI 无法理解美"的常见观点，该公司认为这只是时间问题
2. **写作 = 认知的最高形式**——把写作和审美视为人类认知的"最深处"
3. **不是辅助工具，而是创作主体**——目标是让 AI 本身具备创作能力，而非仅辅助人类创作
4. **人类最高审美标准**——不是"够用就行"，而是追求顶级水准

> 此前所有 JD 关注的"能力"还停留在"解决工程问题"的层面，而本 JD 将目标提升到了**创作和审美**——这是 AGI 更高阶的能力检验场。

### 洞察 3：两个写作方向——文艺 vs 功能

| 维度 | 文艺写作方向 | 功能写作方向 |
|------|-----------|-----------|
| **文体** | 小说、诗歌、散文 | 论文、报告、方案、公文、文案 |
| **核心追求** | **审美价值**——文学的感染力 | **实用性**——信息的有效传递 |
| **质量判断** | 主观（文学品味） | 相对客观（格式规范、信息完整） |
| **评测难度** | **极高**（"好"没有统一标准） | 中高（有约定俗成的标准） |
| **与 Agent 的关系** | 独立创作场景 | 与 [[AI搜索工程]]/[[Agent数据产品工程]] 协同 |
| **用户期望** | 文学价值、情感共鸣 | 准确、规范、高效 |

两个方向的共同点：都需要**极强的文本审美与判断力**——不同的只是审美标准的具体内容。

### 洞察 4："把写得好不好拆解为具体的能力维度"

这是评测体系设计在创作领域的关键挑战：

```
"这个小说不好看"              ← 主观感受，无法操作
    ↓
"人物塑造扁平，缺乏动机层次"
    ↓                                ← 可理解的诊断
"在创作评测中，加入'人物深度'维度"
    ↓                                ← 可复用的评测维度
评分标准：人物是否有明确的动机、成长弧线、内心冲突？
         每项 1-5 分，总分 15 分     ← 可度量的标准
```

| 文体 | 评测维度示例 | 可自动化程度 |
|------|------------|-----------|
| **小说** | 情节逻辑、人物深度、语言表现力、节奏控制 | 低（需人工） |
| **诗歌** | 意象新颖度、韵律感、情感表达、语言凝练度 | 极低（最主观） |
| **论文** | 论点清晰度、论证逻辑、文献引用、格式规范 | 中高（有标准格式）|
| **报告** | 结构完整性、数据准确性、分析深度、可读性 | 中 |
| **公文** | 格式规范、用语正式、信息准确、逻辑清晰 | 高（规范最明确）|

### 洞察 5：与搜索/Agent 方向的协同

JD 特别提到"与搜索/Agent方向协同，在任何环境中给用户提供真正可用的文本"——这说明：

```
搜索场景（JD 4 / JD 11）：需要生成优质的搜索结果摘要
Agent 场景（JD 9 / JD 11）：Agent 输出需要高质量的文本
创作场景（本 JD）：定义什么才是"高质量的文本"
                        ↓
三者的关系：
搜索提供信息 → Agent 执行任务 → 创作定义文本质量标准
                                     ↑
                              本 JD 输出评测标准和高质量数据
```

这意味着创作数据产品经理的工作成果不仅影响"AI 写小说"功能，还会渗透到**所有涉及文本生成的场景**——搜索摘要、Agent 回答、报告生成等。

### 洞察 6："人类千年积累的智慧与审美，是孕育 AGI 的起点"

使命宣言中的这段话，与 JD 12（专业领域）的使命形成了完美的哲学闭环：

```
JD 12：专业领域的深度知识 → 校准模型的专业判断力 → "拓展智能的边界"
本 JD：千年积累的审美智慧 → 校准模型的创作能力 → "让一切从信仰变为现实"
                        ↓
共同信念：人类在各领域的积累不是 AGI 的终点，而是起点。
```

### 洞察 7：跨 JD 全景——十三份 JD 的完整版图

```
产品线（JD 9）     评测数据线（通用/专业/创作）   能力线（JD 10）
    ↑                    ↑                            ↑
AI产品工程          Agent数据产品工程（通用）     Agent能力工程
                    专业领域数据工程（专业）       (Code/安全/Science)
                    AI创作数据工程（创作）★      
    ↑                         ↑                       ↑
           Prompt / Context / Harness 三层工程
           ────────── 基础设施七层 ──────────
```

**三部曲：评测数据产品经理的三种形态**：

| 方向 | JD | 核心问题 | 质量标准来源 |
|------|-----|---------|------------|
| **通用场景** | JD 11 | "Agent 好用吗？" | 用户满意度 |
| **专业领域** | JD 12 | "模型回答得对吗？" | 领域专业标准 |
| **创作领域** | 本 JD | "AI 写得美吗？" | 人类审美最高标准 |

三部曲的难度递增——从相对客观的"好用"，到具有一定标准的"专业"，到**最主观的"审美"**。三者合在一起，构建了从"实用"到"审美"的完整评测光谱。
