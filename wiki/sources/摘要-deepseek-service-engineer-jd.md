# DeepSeek 服务端开发工程师职位描述

> **来源**：DeepSeek 官方招聘信息（2026 年），由 Claude 对话获取并整理。
>
> **团队使命**：与智能共演化，让工程即作品。

---

## 团队定位

服务端工程团队与 [[摘要-deepseek-harness-team-jd|Harness 团队]] 的定位不同：

| 团队 | 聚焦 | 产出 |
|------|------|------|
| **Harness 团队** | Agent 运行时设计（Loop / Tool / Memory / Subagent） | 定义 DeepSeek 对 Harness 的理解 |
| **服务端工程团队** | 底层基础设施与生产服务构建 | 让模型智能稳定、高效地驱动社会进步 |

> 对内：与资深研究员合作，构建用于提升模型智能边界的工程基建
> 对外：深度参与 AI 产品的创新迭代，让智能触达亿万用户

---

## 三个方向

| 方向 | 核心职责 | 对应挑战 |
|------|---------|---------|
| **线上核心服务** | 大模型应用与 API 服务架构设计、性能优化、研发效能工具 | 数千万日活、高吞吐低延迟、可靠性 |
| **Agent 后端** | Agent 执行环境快照、框架集成评测、Agent 数据生成 | Agent 状态持久化、评测自动化、数据反哺模型 |
| **数据仓库** | 数据管道、离线/实时计算、架构稳定性 | 大规模数据处理、流批一体、资源优化 |

---

## 详细职责

### 线上核心服务方向

- 面向数千万日活用户的大模型应用与 API 服务架构设计和持续迭代
- 性能、网络和数据库优化
- 内部研发效能工具与平台搭建
- 不断探索新技术，持续优化服务的可靠性、可扩展性、可观测性、可维护性

### Agent 后端研发方向

该方向与 [[Harness_Engineering]] 关系最为密切，涉及三个独特工程课题：

| 课题 | 说明 | 关联 wiki 概念 |
|------|------|---------------|
| **Agent 执行环境快照系统** | 搭建与维护 Agent 运行时的环境快照机制——将 Agent 的完整执行状态（上下文、工具调用轨迹、中间结果）序列化存储，实现断点续传、回放调试、状态迁移 | [[Agent_Loop]]、[[Checkpoint_Commit]]（类比）、[[Memory_Agent]] |
| **Agent 框架集成与评测** | 接入各类 Agent 框架，支持高效、稳定的评测工作——评测基础设施层，用于 Benchmark Agent 在真实场景的表现 | [[Eval_Harness]]、[[Multi_Agent_System]] |
| **Agent 数据生成** | 与研究员协作，产出高质量 Agent 数据，用以提升模型智能——Agent 执行轨迹作为训练数据反哺模型 | — |

> **关键洞察**：Agent 后端研发方向是 **Harness Engineering 的工程实现层**。三个课题分别对应：执行环境（Runtime Sandbox / State Persistence）、评测（Eval Harness）、数据飞轮（Data Flywheel）。

### 数据仓库方向

- **数据管道**：大规模数据 Pipeline 的设计、开发和落地
- **离线计算**：正确、高效、稳定的数据处理任务
- **实时计算**：流处理系统（监控、实时分析、风控）
- **架构与稳定性建设**：性能提升、链路稳定、资源优化

| 方向 | 独特要求 | 加分项特征 |
|------|---------|-----------|
| 线上核心服务 | 无特殊领域要求 | 大型业务系统经验、开源贡献、竞赛成绩 |
| Agent 后端 | **理解大模型及 Agent 基本原理** | 同上 |

> **值得注意**：Agent 后端方向是唯一明确要求"理解大模型及 Agent 基本原理"的方向，说明它处于 AI 与非 AI 工程的交界处。

## 关键洞察

### 1. 三个方向构成"生产 AI 系统"的三层架构

```
线上核心服务  ← 面向用户的前线（API 网关、应用层）
     ↓
Agent 后端   ← Agent 能力的工程化（执行环境、评测、数据）
     ↓
数据仓库     ← 数据基础（离线/实时计算、存储）
```

三层从"用户触点"到"数据底座"，覆盖了生产级 AI 系统的完整工程链路。

### 2. Agent 执行环境快照是一个新晋工程课题

Agent 执行环境快照（Execution Environment Snapshot）是指将 Agent 运行时的完整状态进行**可序列化、可恢复的持久化**。这与以下需求相关：

- **断点续传**：超长程任务中断后可从快照恢复
- **回放调试**：复现 Agent 的执行轨迹用于 bug 定位
- **状态迁移**：将 Agent 在不同环境间迁移（如从评测环境到生产环境）
- **审计与合规**：记录 Agent 的完整行为轨迹

### 3. Agent 数据生成 ≠ 评测

Agent 数据生成的核心目的不是"评估 Agent 做得好不好"，而是**用 Agent 的执行轨迹作为训练数据反哺模型本身**。这是模型与 Harness 共同进化的关键机制——Harness 不只是消费模型能力，也在生产模型训练素材。

### 4. "工程即作品"的哲学

> **"与智能共演化，让工程即作品。"**

这一团队使命强调工程本身的艺术性与 craftsmanship——不仅仅是"把功能做出来"，而是追求工程的优雅与卓越。
