# 预训练数据工程师职位描述

> **来源**：某大模型公司预训练数据团队官方招聘信息（2026 年），由 Claude 获取并整理。
>
> **核心定位**：预训练数据开采自人类世界全部已有知识所沉积的富矿，是驱动模型强大能力的核心燃料。

---

## 团队使命

预训练数据团队围绕大模型预训练，构建了从数据采集、清洗处理到底层基础设施的完整数据体系，持续为模型训练输送高质量、规模化、多模态的数据燃料，不断拓展模型的世界知识边界。

> **以亿万数据筑就模型智能的坚实底座，用普惠的方式，把属于全人类的财富，重新还给全人类。**

---

## 四个招聘方向

| 方向 | 角色定位 | 核心 Pipeline 环节 |
|------|---------|-----------------|
| **数据采集 Pipeline 方向** | 全网数据选取策略 + 高吞吐分布式采集系统 | 数据源发现 → URL 调度 → 下载 → 流式处理 |
| **语言数据处理方向** | 大规模语料清洗框架 + 离线处理平台 | 清洗框架 → MinHash/向量去重 → 生产流水线维护 |
| **多模态数据方向** | 图像/视频/图文数据处理 + 存储架构 | 数据结构设计 → 清洗/标注链路 → 训练适配供给 |
| **数据基建方向** | 核心数据组件 + 数据湖 + 分布式框架 | KV/消息队列 → 数据湖 → 分布式处理框架 → 性能调优 |

---

## 各方向详情

### 1. 数据采集 Pipeline 方向

**岗位职责：**
- 全网数据选取策略设计开发：链接质量预估、属性聚合、站点抓取配额、数据优先级
- 面向 RAG 等时效场景：时效性种子挖掘、调度与有效性验证
- 全网数据采集环路设计开发：下载、任务调度、流式数据处理、DNS、连通性等核心组件
- 链接规模控制：识别低质站群、重复镜像站点、清洗 URL 冗余参数、规范化链接

**硬性要求：** 计算机/数学相关专业；熟练使用 Python/C++/Rust 之一

**加分项：**
- 数学或信息学竞赛优秀成绩
- 大规模数据采集/爬虫或数据 pipeline 系统研发经验

### 2. 语言数据处理方向

**岗位职责：**
- 预训练语料清洗框架研发：构建稳定、高效、可扩展的大规模数据处理平台
- 核心语料清洗能力：MinHash 去重、向量去重等关键环节
- 日常生产流水线维护：增量处理、任务调度、数据统计、质量回归、线上问题排查
- 系统性能、稳定性和可观测性优化

**硬性要求：** 计算机/AI 相关专业；熟练使用 Python + pandas/spark/duckdb 等工具链

**加分项：**
- 对世界有好奇心，对模型行为有独特理解
- 对模型能力边界有敏锐判断
- 优秀的跨团队沟通能力
- 大规模数据清洗 pipeline 落地经验（TB 级别 tokens）

### 3. 多模态数据方向

**岗位职责：**
- 大规模多模态数据（图像、视频、图文结合）的数据结构设计、存储架构规划与全生命周期管理
- 多模态数据清洗与标注链路的工程优化：结合内容理解技术提升去重/结构化/质量筛选/打标分析效率
- Pretrain/SFT/RL 不同阶段的数据存储方案与数据结构设计
- 数据可视化分析体系搭建：挖掘数据特征与模型训练效果的关联

**硬性要求：** 计算机基础扎实；熟悉 Python，了解 PyTorch；具备系统性能分析与调优能力

**加分项：**
- 大规模分布式数据处理系统研发与性能优化经验
- 高并发、异步编程经验，完整的大规模数据链路开发与调优经历
- 了解多模态模型训练与部署框架（如 Megatron、vLLM 等）
- 算法竞赛（NOI/IOI/ICPC/CCPC 等）优异成绩

### 4. 数据基建方向

**岗位职责：**
- 键值数据库、消息队列等核心数据组件的开发、维护与调优
- 面向多模态海量数据的数据管理与数据湖方案设计落地
- 分布式数据处理框架设计开发，支撑 PB 级数据处理
- CPU/内存/网络调优，提升吞吐与资源利用率
- 提供易用的 API、库与工具，支撑数据探索、加工与规模化使用

**硬性要求：** 计算机相关专业；扎实 Rust/C++/Python 编程能力；扎实的工程能力与系统功底

**加分项：**
- 大规模存储、分布式 KV、数据库或数据湖、分布式数据处理框架的设计/研发/运维经验
- 高水平系统会议论文

## 关键洞察

### 1. 数据工程是三层工程模型的基础支撑层

预训练数据团队的工作横跨三层工程模型：

| 工程层 | 对应工作 |
|-------|---------|
| **Context Engineering** | 语料清洗、去重、质量筛选、多模态数据组织——决定"喂给模型什么数据" |
| **Harness Engineering** | 数据采集环路、分布式处理框架、KV/消息队列、调度与容错——决定"数据怎么跑" |
| **底层系统** | 数据湖、对象存储、分布式文件系统——决定"数据存在哪、怎么存" |

### 2. 从数据采集到模型训练的完整闭环

```
互联网/数据源
    ↓
数据采集 Pipeline（调度/下载/流式处理）
    ↓
语言数据处理 + 多模态数据处理（清洗/去重/标注/筛选）
    ↓
训练框架（Pretrain / SFT / RL）
    ↓
模型 → 反馈信号 → 优化数据策略
```

### 3. 从"Pipeline"到"Platform"的技术纵深

四个方向构成了从**数据获取**到**基建支撑**的完整技术光谱：
- 采集 Pipeline：离数据源最近，考验大规模分布式系统 + 策略设计
- 语言数据处理：离模型最近，考验清洗算法 + 生产流水线工程
- 多模态数据：离多样性最近，考验结构化 + 全生命周期管理
- 数据基建：离基础设施最近，考验系统底层功底 + 数据库/存储

### 4. "蒙尘的富矿"隐喻

> "寻找蒙尘的富矿，并淘洗炼制成可用的智能燃料"

这反映了预训练数据工程的核心挑战：高质量数据往往隐藏在低质量、高噪声的大规模原始数据中，需要精细的策略（采集选择）和工程（清洗加工）才能释放其价值。
