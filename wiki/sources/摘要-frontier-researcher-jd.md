# Frontier（持续学习/自进化/新范式）研究员职位描述

## 来源概览

- **职位**: Frontier（持续学习/自进化/新范式）研究员
- **公司信号**: DeepSeek
- **团队使命**: "我们与 AGI 的距离也许不只是 scaling……一起驱散 AGI 上空仍然存在的乌云，推动后续范式的到来"
- **核心关键词**: 持续学习、自进化、下一代架构、下一代学习算法、范式转移

---

## 关键洞察

### 洞察 1：这是整个堆栈的"种子层"

所有之前的 JD 关注的都是**在当前范式下做事**——产品、工程、评测、数据。本 JD 关注的却是**当前范式本身可能有缺陷**，需要新的范式。

```
                  ┌──────────────────────────────────┐
                  │  Frontier 研究（新范式探索）       │ ← ★ 本 JD
                  │  持续学习 / 自进化 / 新架构 / 新算法 │      范式层
                  └──────────┬───────────────────────┘
                             ↓ 新范式 → 一切重来
                  ┌──────────────────────────────────┐
产品/评测/数据层 │  AI产品工程 / Agent数据产品工程     │
                  │  专业领域/创作/情感数据工程        │      范式内优化
                  └──────────┬───────────────────────┘
                             ↓ 能力 + 框架
                  ┌──────────────────────────────────┐
   工程/基础设施层 │  Harness / Prompt / Context       │
                  │  训练/计算/集群/存储/硬件/设施/沙箱│     范式内执行
                  └──────────────────────────────────┘
```

| 层次 | 角色 | 关注 | 时间尺度 |
|------|------|------|---------|
| **范式层** | Frontier 研究员 | **下一个范式是什么？** | 年-十年 |
| **产品/数据层** | 产品经理/数据PM | **在当前范式下怎么做到最好？** | 周-月 |
| **能力层** | Agent 能力工程师 | **怎么让模型在当前范式下更强？** | 月 |
| **工程层** | 各类工程师 | **怎么在当前范式下高效执行？** | 天-周 |

### 洞察 2："我们与 AGI 的距离也许不只是 scaling"

这是整个知识库中最深刻的命题——它直接挑战了当前 AI 领域的主流叙事：

```
主流叙事（Scaling Law）：
    更多数据 + 更大模型 + 更多算力 = 更强的能力 → AGI
        ↓
本 JD 的立场：
    Scaling 可能不够。还差"最后的拼图"——下一个范式。
        ↓
这意味着：
    1. 当前范式（Transformer + Scaling + RLHF）可能不是终局
    2. AGI 可能需要架构/算法层面的根本性突破
    3. 研究需要承担"范式转移"的风险——大多数方向不会成功
```

> **"愿意承担研究风险，敢于尝试，能对困难问题进行长期冲击"** ——这是十四份 JD 中唯一明确要求承担高失败风险的角色。

### 洞察 3：四大研究方向——持续学习、自进化、新架构、新算法

JD 列出了四个明确的方向：

| 方向 | 核心问题 | 如果成功 | 当前范式的局限 |
|------|---------|---------|--------------|
| **持续学习** | 模型如何在不断学习中不遗忘旧知识？ | 模型持续演进，无需完整重训 | 预训练是一次性的；微调会灾难性遗忘 |
| **自我进化** | 模型如何自己改进自己？ | 模型通过自我反思和实验不断进步 | 当前依赖人工标注的 RLHF/DPO |
| **下一代架构** | Transformer 之后是什么？ | 更高效、更强大、更长上下文 | Attention 的 O(L²) 复杂度、固定计算图 |
| **下一代学习算法** | 比 RLHF/DPO 更好的训练方法？ | 更高效、更稳定、更通用的学习方法 | RLHF 的不稳定性、Reward Hacking |

### 洞察 4：加分项——完整的新范式探索知识图谱

JD 列出了 9 个加分的研究方向：

```
持续学习（Continual Learning）       ← 学而不忘
    ↓
自我进化（Self-Evolution）          ← 自我改进
    ↓
Long Horizon Agent                 ← 长周期自主 Agent
    ↓
Agentic Memory                     ← Agent 记忆架构
    ↓
Self Distillation                  ← 自我知识蒸馏
    ↓
PEFT（Parameter-Efficient Fine-Tuning）← 高效微调
    ↓
Auto Research                      ← 自动化研究
    ↓
AI for AI                          ← 用 AI 改进 AI
    ↓
Memory Architecture                ← 新型记忆结构
    ↓
Optimizer                          ← 新型优化器
```

这些方向共同指向一个愿景：**让模型具备持续自我改进的能力**——一旦实现，当前所有数据产品经理/能力工程师的工作方式都可能被颠覆。

### 洞察 5：与所有其他 JD 的关系——范式转移的"毁灭性创造"

本 JD 是所有其他 JD 的**基础层**——因为如果新范式出现，所有在当前范式下建立的知识体系都需要重新审视：

```
当前范式：Transformer + Pre-training + RLHF/DPO
    ↓ 成功
    新产品范式：如果新架构替代了 Transformer……
        ↓
    → 训练推理框架需要重写
    → 算子/通信/编译器需要适配新硬件
    → 超算集群架构可能改变
    → 评测体系需要重新设计
    → 产品体验有新的可能
```

但反过来也成立——当前范式下所有层的实践者，为本层提供了：

```
数据产品经理（JD 11-14）：提供当前范式的能力边界数据
    → Frontier 研究员知道当前范式在哪还不够好→寻找新范式

Agent 能力工程师（JD 10）：发现当前范式的"天花板"
    → Frontier 研究员知道什么方向值得探索

工程/基础设施团队（JD 1-8）：提供研究所需的基础设施
    → Frontier 研究员需要算力和数据来验证新想法
```
