# Agent 实例生命周期管理

## 定义

Agent 实例生命周期管理是指 **AI Agent 从创建到销毁全过程的管控体系**——包括实例的创建、状态维持、调度执行、休眠/恢复、销毁与清理。与传统微服务实例不同，Agent 实例是**有状态**的（持有 Context Window / Memory / 执行轨迹），这使得其生命周期管理远比无状态服务复杂。

> 核心矛盾：Agent 需要"记住"上下文因此有状态，但平台需要弹性因此想无状态。

## Agent 实例的生命周期阶段

```
        ┌──────────────────┐
        │    创建 (Create)   │
        │  ─ 选择 Agent 类型  │
        │  ─ 分配 Tenant 身份 │
        │  ─ 初始化 Memory   │
        └────────┬─────────┘
                 ▼
        ┌──────────────────┐
        │    就绪 (Ready)    │ ←── 资源已分配但未开始工作
        │  ─ 等待用户输入     │
        └────────┬─────────┘
                 ▼ (用户触发)
        ┌──────────────────┐
        │    运行 (Running)  │
        │  ─ Agent Loop 执行 │ ←── 最复杂的阶段
        │  ─ Tool Call 编排  │     (含暂停/恢复)
        │  ─ 上下文动态变化   │
        └────────┬─────────┘
                 ▼ (完成/超时/错误)
        ┌──────────────────┐
        │    终止 (Terminated)│
        │  ─ 正常完成         │
        │  ─ 超时终止         │
        │  ─ 错误终止         │
        │  ─ 用户取消         │
        └────────┬─────────┘
                 ▼
        ┌──────────────────┐
        │    清理 (Cleanup)   │
        │  ─ 释放计算资源     │
        │  ─ 持久化关键状态   │
        │  ─ 归档执行轨迹     │
        │  ─ 关闭连接        │
        └──────────────────┘
```

## 关键设计维度

### 1. 状态管理策略

| 策略 | 说明 | 适用场景 | 复杂度 |
|------|------|---------|-------|
| **全内存态** | Agent 状态全程在内存中，销毁即丢失 | 短任务、非关键对话 | ⭐ |
| **Checkpoint 持久化** | 关键节点序列化到外部存储 | 长任务、需恢复的场景 | ⭐⭐⭐ |
| **Event Sourcing** | 所有操作记录为事件流，可重放 | 审计合规、Debug、Time-travel | ⭐⭐⭐⭐⭐ |
| **Hybrid** | Working Memory 在内存 + Long-term 定期持久化 | 大多数 Production 场景（推荐） | ⭐⭐⭐ |

### 2. Agent 实例隔离级别

| 隔离级别 | 资源分配 | 启动时间 | 安全隔离度 | 适用场景 |
|---------|---------|---------|----------|---------|
| **进程级容器** | 独立 Container/Pod | ⭐⭐（秒级） | ⭐⭐⭐⭐⭐ | 高风险 Agent、企业定制 |
| **线程级轻量** | 共享进程独立 Session | ⭐⭐⭐⭐⭐（毫秒级） | ⭐⭐⭐ | 标准对话 Agent |
| **Serverless** | 按需分配，冷启动 | ⭐⭐（秒级，有冷启动） | ⭐⭐⭐⭐ | 低频 Agent、弹性伸缩 |
| **WebWorker** | 浏览器端运行 | ⭐⭐⭐⭐⭐ | ⭐⭐ | 边缘 Agent、客户端 Agent |

### 3. 运行时调度

```
用户请求到达
  → 调度器判断：
      ├── 是否有已就绪的 Agent 实例？→ 复用
      ├── 是否需要新建？→ 检查资源配额
      │     ├── 当前租户未超限 → 创建新实例
      │     └── 超限 → 排队或拒绝
      └── 是否需要从 Checkpoint 恢复？→ 加载持久化状态
  → 路由到目标 Agent Runtime
  → 执行
  → 完成后决定：保持就绪 / 休眠 / 销毁
```

#### 调度策略对比

| 策略 | 说明 | 适合场景 |
|------|------|---------|
| **Least-Recently-Used** | 优先调度空闲最久的 Agent | 资源复用最大化 |
| **Affinity** | 同一用户绑定到同一实例 | 对话连续性优先 |
| **Isolation-first** | 每个请求独立新实例 | 安全和隔离优先 |
| **Cost-aware** | 根据 Token/计算预算调度 | 成本控制优先 |

### 4. 伸缩策略

| 策略 | 触发条件 | 效果 | 注意事项 |
|------|---------|------|---------|
| **水平扩展** | 就绪队列长度 > 阈值 | 增加 Agent Runtime Pod 数量 | 有状态 Agent 需外部化 Session |
| **垂直扩展** | 单个 Agent Context 超阈值 | 升级更大 Context Window 的模型 | 成本线性增长 |
| **按租户缩** | 特定租户活跃度下降 | 回收该租户空闲实例 | 需保留 Session 元数据 |
| **预测性扩** | 基于历史流量预测 | 提前调度就绪实例 | 减少冷启动延迟 |

### 5. 超时与终止策略

| 超时类型 | 默认值 | 行为 | 可配置性 |
|---------|-------|------|---------|
| **会话超时** | 30 分钟无交互自动休眠 | 持久化后释放资源 | 按租户/Agent 类型可配 |
| **执行超时** | 单次 Agent Loop 10 分钟 | 中断执行，返回部分结果 | 按任务类型可配 |
| **空闲超时** | 5 分钟 | 标记为可回收 | 全局阈值 |
| **总生命周期超时** | 24 小时 | 强制销毁，无论是否活跃 | 安全策略固定 |

### 6. 休眠与恢复

对于长时间多轮对话的 Agent，引入休眠/恢复机制是效率关键：

```
休眠触发 → 序列化 Context Window 快照
         → 持久化到外部存储 (Redis / S3)
         → 释放计算资源
         → 标记 Agent 为 "休眠" 状态

恢复触发 → 用户新消息到达
         → 调度器查找该用户的休眠实例
         → 加载 Context 快照到新 Runtime
         → 恢复执行
```

| 恢复方式 | 延迟 | 实现复杂度 | 说明 |
|---------|------|-----------|------|
| **热恢复** | <100ms | ⭐⭐⭐ | 保持 Runtime 进程，只在内存和磁盘间交换 Context |
| **温恢复** | 1-5s | ⭐⭐ | 保留镜像，重新分配 Pod 然后加载快照 |
| **冷恢复** | 5-30s | ⭐ | 完全新建 + 从 S3 加载 + 重放 Event Log |

## 多租户环境中的特殊性

| 管理维度 | 单租户 | 多租户 |
|---------|-------|-------|
| **实例命名空间** | 扁平 | 按 tenant_id 分层隔离 |
| **资源配额** | 全局池 | 按租户的 Token/Pod 配额独立核算 |
| **可见性** | 管理员可见全部 | 租户管理员只可见自己租户的实例 |
| **计费** | 不需要 | 按租户维度的实例运行时长/Token 消耗计费 |
| **SLA** | 统一 | 不同租户可配置不同 SLA（企业版 vs 免费版） |
