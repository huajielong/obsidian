## 定义

温度参数（Temperature）是 LLM 推理中控制输出随机性的核心参数。它通过缩放 Softmax 层的 logits 分布来影响 Token 采样的概率：温度越低，高概率 Token 的优势越明显（趋向确定性）；温度越高，低概率 Token 被选中的机会越大（趋向多样性）。

## 关键信息

### 取值范围

| 温度 | 确定性 | 多样性 | 推荐用途 |
|:---:|:-----:|:-----:|---------|
| **0.0** | **100%** | 无 | 事实问答、代码生成、结构化输出 |
| 0.1~0.3 | 高 | 低 | 翻译、摘要、少量变体 |
| 0.5~0.7 | 低 | 中高 | 创意写作、对话、头脑风暴 |
| 0.8~1.0 | 很低 | 高 | 诗歌、故事、创意探索 |
| >1.0 | 不可控 | 极高 | 极少使用，可能产生无意义内容 |

> **API 限制**：Ollama / OpenAI 限 0.0~2.0，超出会被截断。理论上无上限，但 >2.0 时输出基本乱码。

### 实验发现

1. **温度 = 0（贪婪解码）**：20/20 次输出完全一致，行为完全确定
2. **温度 0.1 即开始变化**：5 次中已有 2 种不同输出
3. **温度 0.3+ 完全多样**：5/5 各不相同
4. **温度 1.5 语言混入**：英文单词直接出现在中文上下文中
5. **长度控制失效**：温度 0.7 时同一 Prompt 的 Token 极差可达 **333（28 倍）**

### 注意事项

- **温度 > 0 时输出长度不可预测**：同一 Prompt 的 Token 数波动可达 28 倍
- **指令遵循能力随温度升高下降**：长度约束在高温度下基本失效
- **生产环境建议**：依赖格式的场景务必使用 temperature=0
- **需要多样性时推荐**：temperature=0.7 + top_p=0.9 组合使用
