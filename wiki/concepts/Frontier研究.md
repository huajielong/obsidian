# Frontier 研究（Frontier Research）

## 定义

Frontier 研究是**以超越当前 AI 范式为目标，通过批判性思考、前沿探索和高风险实验，寻找通向 AGI 的下一个范式突破的系统化研究实践**。它解决的核心问题是：

> **如果 Scaling Law 不够到达 AGI，我们还缺什么？**

Frontier 研究是整个知识体系中最"上层"也最"底层"的一层——最上层是因为它关注未来最前沿的可能性，最底层是因为它探索所有层当前依赖的范式本身是否站得住脚。

> **所有工程都是在当前范式内做事，Frontier 研究在寻找让当前范式过时的东西。**

### 与所有其他概念的关系

Frontier 研究与其他概念之间不是"上下游"关系，而是**范式→范式内实践**的关系：

```
当前范式（Transformer + Scaling + RLHF）
    │
    ├── 范式内优化层：Harness / Prompt / Context ……
    ├── 范式内执行层：训练工程 / 计算引擎 / 超算集群 ……
    ├── 范式内数据层：评测数据 / 领域数据 / 创作数据 ……
    ├── 范式内产品层：AI 产品工程
    └── 范式内能力层：Agent 能力工程
            │
            ↓ 如果新范式出现，以上所有层都可能需要"重置"
            ↓
    Frontier 研究（探索新范式）
```

## 四大研究方向

### 1. 持续学习（Continual Learning）

**核心问题**：模型如何在持续学习新知识的同时，不遗忘已学到的旧知识？

#### 持续学习的核心技术挑战

```
灾难性遗忘（Catastrophic Forgetting）：
    学习新任务 → 网络权重被覆盖 → 旧任务能力断崖式下降
        ↓
    需要"弹性权重"——哪些权重重要、不应改变？

稳定性-可塑性困境（Stability-Plasticity Dilemma）：
    太稳定 → 学不进新东西
    太可塑 → 学新忘旧
        ↓
    需要动态平衡机制
```

#### 与当前范式的关系

| 当前做法 | 持续学习的目标 | 差距 |
|---------|--------------|------|
| 预训练一次，最多微调 | **模型持续演进** | 当前无法在学习过程中保持旧能力 |
| 定期完整重训 | **增量更新** | 完整重训消耗巨大能源 |
| 在固定数据集上训练 | **开放式学习** | 当前模型不能自己选择学什么 |
| 人工决定训练数据 | **自主学习** | 模型不能自主判断知识缺口 |

### 2. 自我进化（Self-Evolution）

**核心问题**：模型如何不依赖人类的标注和反馈，自己改进自己？

#### 自我进化的光谱

```
L1：自我蒸馏（Self-Distillation）
    模型用自己的输出训练自己 → 收敛到自身能力的极致
    ↓
L2：自我反思（Self-Reflection）
    模型评估自己的输出 → 发现错误 → 改进
    ↓
L3：自我探索（Self-Exploration）
    模型主动探索未知领域 → 发现新知识 → 整合
    ↓
L4：自我目标设定（Self-Goal Setting）
    模型判断自己"应该会什么" → 设定学习目标 → 达成
    ↓
L5：完全自主进化（Autonomous Evolution）
    模型自我导向、自我调整、自我超越 —— 接近 AGI 的核心能力
```

#### 与当前范式的关系

```
当前：RLHF = 人类标注偏好 → 模型学习偏好
    ↓ 缺点：依赖昂贵的人工标注，模型不会自己判断"对错"

目标：Self-Evolution = 模型自己判断"好坏" → 自己改进
    ↓ 挑战：如何防止模型 Self-Evolution 到错误的方向？
    ↓       如何让模型对自己的能力有准确的感知？
```

### 3. 下一代模型架构（Next-Gen Architecture）

**核心问题**：Transformer 之后，深度学习的基础架构应该是什么？

#### Transformer 的已知局限

| 局限 | 表现 | 新架构的追求 |
|------|------|------------|
| **O(L²) 计算复杂度** | 长序列不可承受 | **线性/亚线性复杂度** |
| **固定计算图** | 每个 token 花相同的计算量 | **动态路由**——给"难" token 更多计算 |
| **注意力机制的"滩涂"** | 长上下文中注意力弥散 | **更高效的记忆检索** |
| **无法增量更新** | 学习新知识需完整重训 | **可增量扩展的架构** |
| **硬件利用不均** | Attention 是访存密集型 | **计算密集型替代方案** |

#### 潜在新架构方向

```
状态空间模型（SSM / Mamba）：
    线性复杂度、长序列友好 → 但缺乏注意力机制的灵活性
    
混合架构（Hybrid）：
    注意力 + SSM 结合 → 兼顾效率和灵活性
    
动态架构（Dynamic）：
    计算图动态调整 → 按需分配计算资源
    
神经符号系统（Neuro-Symbolic）：
    连接主义 + 符号推理 → 精确性和泛化性的统一
    
Liquid / 连续时间网络：
    连续时间动力学 → 更接近生物神经元的运作方式
```

### 4. 下一代学习算法（Next-Gen Learning Algorithm）

**核心问题**：比 RLHF 更高效、更稳定、更通用的训练方法是什么？

#### 当前学习算法的局限

| 算法 | 局限 | 新算法追求 |
|------|------|-----------|
| **RLHF/PPO** | 不稳定、需要大量人工标注、Reward Hacking | **更稳定的偏好学习** |
| **DPO** | 假设偏好是静态的，不考虑多轮交互 | **动态偏好建模** |
| **SFT** | 依赖高质量人工数据，无法自我提升 | **自主学习方法** |
| **RL** | 奖励设计困难、探索效率低 | **高效的探索策略** |

#### 潜在新算法方向

```
自我监督学习（Self-Supervised）的进阶版：
    不只是"预测下一个 token"——也许有更好的预训练目标

过程奖励模型（Process Reward Model）：
    不是只奖励最终结果，而是奖励推理过程的每一步
    
内在动机（Intrinsic Motivation）：
    模型自己产生"好奇心"——主动探索知识盲区
    
元学习（Meta-Learning）：
    学习如何学习——在新任务上快速适应
    
世界模型（World Model）：
    在学习语言的同时学习物理世界的因果结构
```

## 核心能力

### 1. 批判性思考

JD 要求"擅长批判性思考，对前沿的工作有自己的批判性思考"——在 AI 研究飞速发展的环境中，这意味着：

```
跟随潮流：看了新的 Paper → 接受结论
批判性思考：看了新的 Paper → 问：
    - 实验设计是否有缺陷？
    - 结论是否只在特定条件下成立？
    - 这个方法的核心洞察可以推广到什么范围？
    - 如果这个方法是对的，它预示了什么？限定了什么？
```

### 2. 对当前范式的深刻理解

JD 要求"对当前范式有深刻的理解，有强大的洞察力"——要寻找新范式，必须先深刻理解当前范式为什么还不够：

```
不理解当前范式的人：        觉得当前范式"完美"
有些理解的人：              觉得当前范式"有改进空间"
深刻理解的人（目标群体）：   知道当前范式的根本局限在哪里
                           知道为什么这些局限不是"修修补补"能解决的
                           知道哪些方向值得探索、哪些方向是死胡同
```

### 3. 承担研究风险的意愿

> "愿意承担研究风险，敢于尝试，能对困难问题进行长期冲击"

这是最具筛选性的要求——意味着：

```
大多数研究项目不会成功
大多数投入不会立即产生结果
你可能在同一个问题上花数年时间
但你相信这是值得的——因为一旦突破，一切都不一样了
```

## 关键挑战

### 1. 如何判断"下一个范式"的方向？

这是最根本的挑战——在众多可能的方向中，如何选择？

```
策略一：Top-Down
    从"AGI 还需要什么能力"出发
    → 确定当前范式的根本局限
    → 寻找能突破这些局限的方法

策略二：Bottom-Up
    从当前研究中的"异常现象"出发
    → 某些实验为什么不符合理论预期？
    → 这些异常是否暗示了新的可能性？

策略三：跨领域迁移
    从其他领域（神经科学、物理学、复杂性科学）寻找启发
    → 生物大脑是怎么解决持续学习的？
    → 物理学的"相变"概念能否应用于学习？
```

### 2. 长期研究与短期成果的矛盾

```
公司盈利压力：今天的模型需要今天改进 → 短期的、可预测的研究
Frontier 研究：不知道什么时候能出成果 → 长期的、高风险的研究
    ↓
解决方案（JD 中暗示）：
    公司愿意在"探索"和"利用"之间保持平衡
    Frontier 研究不需要用短期成果证明自己
```

### 3. 新范式的"鸡和蛋"问题

```
新范式在没有证明之前，很难获得资源
    ↓
没有足够的资源，很难验证新范式
    ↓
解决方案：
    小规模验证（在小模型/小数据上验证核心想法）
    理论论证（证明想法在理论上是可行的）
    类比论证（借鉴其他领域的成功范式转移案例）
```

## 工程哲学

### 1. "Scaling 不够"——最根本的信念

这是整个知识库中**最不"工程师"的工程信念**：

> **我们可能不是缺算力，而是缺想法。** 如果 Scaling 能到 AGI，那 AGI 就只是一个工程问题。但如果 Scaling 不够，那就需要科学上的突破——而突破来自观点、来自实验、来自愿意挑战主流的勇气。

### 2. "驱散 AGI 上空的乌云"——对未知的谦逊

> "一起驱散 AGI 上空仍然存在的乌云"

这句话展现了一种对未知的谦逊——当前范式还有许多"乌云"，我们没有完全理解：

```
什么"乌云"？
    1. 推理能力：模型能正确推理吗？还是只是模式匹配？
    2. 泛化能力：模型能真正超越训练数据吗？
    3. 自我意识：模型有"自我"的概念吗？
    4. 开放性：模型能处理完全未知的情况吗？
    5. 持续学习：模型能不遗忘地持续进步吗？
```

### 3. "下一个范式"——对革命性创新的追求

不追求增量改进，追求**范式转移**：

```
增量改进：把 Transformer 的效率提高 10%
    ↓ 有价值，但不是"范式转移"
范式转移：找到一种完全不同于 Transformer 的架构
    ↓ 更难、更有风险，但一旦成功，可能让 AGI 真正到来
```

## 知识冲突

### Scaling 是否足够？
- **Scaling Law 派**：继续 scaling + 更多高质量数据 = AGI，不需新范式
- **范式转移派**：Scaling 正在遇到收益递减，需要根本性突破
- **调和**：两条腿走路——在 Scaling 路线上继续推进的同时，也在寻找下一范式

### 应该投入多少资源到 Frontier 研究？
- **实用派**：将大部分资源投入到当前范式下的工程优化和产品迭代
- **冒险派**：必须划拨足够资源到高风险研究，否则可能错过下一个范式
- **调和**：公司规模足够时，可以并行推进——"探索"和"利用"双线

### 持续学习 vs 完整重训——哪个路线正确？
- **持续学习派**：模型应该像人类一样持续积累知识，不应该依赖于周期性的完整重训
- **完整重训派**：周期性的完整重训更可靠、更可控、效果更好
- **调和**：可能最终方案是两者的混合——核心架构不变的基础上的知识增量更新
