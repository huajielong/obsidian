# AI 创作数据工程（AI Creative Data Engineering）

## 定义

AI 创作数据工程是**将人类审美标准转化为可操作的评测体系与数据管线，系统性地驱动模型在文学创作和实用写作领域能力持续提升的工程实践**。它解决的核心问题是：

> **如何让 AI 从"能写出通顺的文本"进化到"写出有审美价值的创作"。**

AI 创作数据工程是 [[Agent数据产品工程]]（评测数据方法论）在**创作与审美领域**的展开——与 [[专业领域数据工程]]（知识精确性）并列，构成评测数据工程的两个专业方向。

> **专业领域数据工程 ≈ 校准模型的"对错"**
> **AI 创作数据工程 ≈ 校准模型的"美丑"**

## 两大创作方向

### 1. 文艺写作方向

**核心问题**：如何让 AI 创作出有文学价值的虚构作品？

#### 覆盖文体

| 文体 | 核心挑战 | 评测维度（示例） | 当前能力水平 |
|------|---------|----------------|------------|
| **小说** | 情节架构、人物塑造、叙事节奏 | 情节逻辑性、人物深度、语言感染力、悬念设计 | 基础叙事可用，长篇架构弱 |
| **诗歌** | 意象创造、韵律、情感表达 | 意象新颖度、语言凝练度、情感真实度、形式美感 | 最早期，偶有闪光 |
| **散文** | 个人化表达、观察深度 | 视角独特性、情感真切度、语言风格 | 中等，但与人类差距大 |
| **剧本** | 对话设计、场景转换、冲突构建 | 对白自然度、场景衔接、戏剧张力 | 早期探索阶段 |

#### 文艺写作评测的特殊挑战

```
技术类评测（代码/知识）：
    对错清晰 → 自动化验证可行 → 评测相对简单

创作类评测（小说/诗歌）：
    对错不清晰 → 缺乏统一标准 → 评测最困难
        ↓
    解决方案：
    1. 建立"审美共识"——由多位文学专家共同标注标杆
    2. 拆解维度——将"写得好"拆解为可讨论的具体维度
    3. 层次化评分——不是"好/不好"，而是分项打分
    4. 保留多样性——承认多种创作风格都可以是"好"的
```

### 2. 功能写作方向

**核心问题**：如何让 AI 在各种实用场景中写出真正"可用"的文本？

#### 覆盖文体

| 文体 | 核心要求 | 评测维度 | 与 Agent 协同场景 |
|------|---------|---------|----------------|
| **论文** | 论点清晰、论证严谨、格式规范 | 结构完整性、论据充分性、引用准确性 | Agent 辅助研究写作 |
| **报告** | 数据准确、分析深入、可读性强 | 数据支撑、分析逻辑、图表规范 | Agent 数据分析后输出报告 |
| **方案** | 目标明确、路径可行、考虑周全 | 完整性、可行性、风险考量 | Agent 规划任务输出方案 |
| **公文** | 格式规范、用语正式、信息准确 | 格式合规、用语规范、信息完整 | Agent 自动生成公文 |
| **文案** | 吸引力、信息传递、品牌一致 | 感染力、信息密度、调性一致 | Agent 营销内容生成 |

#### 功能写作与搜索/Agent 的协同

```
用户搜索信息（[[AI搜索工程]]）
    ↓
Agent 获取信息并执行任务（[[Agent数据产品工程]]）
    ↓
Agent 需要输出文本
    ↓
本层（AI创作数据工程）提供：
    ├── 文本质量标准（什么算是"好"的写作）
    ├── 评测维度（如何衡量文本质量）
    └── 高质量数据（模型学习写作的范本）
```

## 三大核心职责

### 1. 优化方向判断与效果评测

**核心问题**：模型的创作能力当前处在什么水平？下一步该往哪个方向优化？

#### "导航图"方法

```
模型当前创作能力水平
    ↓ 评测定位
地图上的当前位置（如：叙事流畅但人物扁平）
    ↓ 方向判断
下一个里程碑（如：提升人物塑造深度）
    ↓ 路径设计
具体的数据和训练方案
    ↓ 效果验证
新一轮评测确认是否到达里程碑
```

#### 文体特异化的评测维度

| 通用维度 | 小说版 | 诗歌版 | 报告版 | 公文版 |
|---------|-------|-------|-------|-------|
| **准确性** | 情节逻辑一致 | 意象逻辑可感 | 数据准确 | 信息无误 |
| **完整性** | 故事有始有终 | 意境完整 | 覆盖所有要点 | 要素齐全 |
| **语言** | 表现力、风格 | 凝练、韵律 | 清晰、专业 | 正式、规范 |
| **结构** | 起承转合 | 节奏控制 | 层次分明 | 格式标准 |
| **创意** | 新颖性、独特性 | 意象原创性 | 分析视角新颖 | — |

### 2. 定义"理想态"标准

**核心问题**：对于每一种文体，什么算是"最好"的输出？

#### 理想态的层次

```
L1：正确无误     语法正确、事实准确、格式规范
    ↑
L2：流畅可读     表达通顺、逻辑清晰、阅读体验好
    ↑
L3：专业水准     符合领域标准、信息组织高效、风格恰当
    ↑
L4：审美价值     语言有感染力、结构有匠心、风格有辨识度
    ↑
L5：杰作级       超越当前标准、开辟新的表达可能
```

不同文体对层次的要求不同：

| 文体 | 最低可接受 | 理想目标 | 评测难度 |
|------|----------|---------|---------|
| **公文** | L1（正确无误） | L3（专业水准） | 中 |
| **报告** | L2（流畅可读） | L3（专业水准） | 中高 |
| **文案** | L2（流畅可读） | L4（审美价值） | 高 |
| **散文** | L3（专业水准） | L4（审美价值） | 很高 |
| **小说** | L2（流畅可读） | L4-L5（审美-杰作） | 极高 |
| **诗歌** | L3（专业水准） | L5（杰作级） | 最高 |

### 3. 制定数据质量标准与生产规范

**核心问题**：什么样的数据才能让模型真正"学会写作"？

#### 数据质量的三重标准

```
内容标准：
    ├── 文本本身是否高质量（符合目标层次）
    ├── 是否适合作为模型学习的范本
    └── 多样性是否充分（不同风格、难度、体裁）

标注标准：
    ├── 评测标注是否准确（多标注者一致性）
    ├── 反馈标注是否有建设性（指明了具体问题）
    └── 标注规范是否清晰可复现

实用标准：
    ├── 数据是否能直接用于训练
    ├── 是否能支撑评测体系的运行
    └── 是否能覆盖模型当前的能力短板
```

## 核心技术能力

### 1. 文本审美与判断力

这是最核心的能力——JD 要求：

> "在文学创作或实用文本写作领域有长期、扎实的实践，对文字有极高的敏感度，能对文本质量做出精准判断"

审美不是"我觉得好"而是：

```
看见：能发现文本中的细微差异（同样的意思，为什么这个表达更好？）
分析：能将"好"的原因说清楚（不是因为职位高低，而是因为……）
拆解：能将审美感受转化为可操作的维度（不是"读着怪怪的"，而是"这句话的修辞与上下文语境不协调"）
传授：能让其他人理解和复现你的判断标准
```

### 2. 跨文体能力

JD 要求同时覆盖文艺写作和功能写作——这两者需要不同的审美标准：

```
文艺写作的核心：语言的表现力、情感的真实度、形式的创新性
    ↓
功能写作的核心：信息的准确性、结构的逻辑性、格式的规范性
    ↓
共同点：都需要文本判断力，但判断的维度不同
```

### 3. 审美判断→评测维度的转化能力

这是"数据产品经理"最关键的技能——将主观感受客观化：

```
感性的判断                        理性的评测
──────────────────────────────────────
"这段小说读起来没感觉"      →    "人物情感描写缺乏细节支撑"
"这个报告写得太散了"        →    "报告缺少分段的主题句，结构层次不清"
"这个文案不够吸引人"        →    "文案缺少'钩子'(Hook)，前三句没有建立阅读动机"
"这首诗不像诗"              →    "分行和节奏缺乏设计，语言密度不足"
```

## 关键挑战

### 1. 审美的主观性 vs 评测的可复现性

这是 AI 创作数据工程**最大的核心挑战**：

```
问题：文学审美是高度个人化的——A 觉得好，B 可能觉得不好
    如何在主观审美上建立客观的评测体系？

解法：
1. 多专家共识：不是一个人说了算，而是建立"专家共识"数据库
2. 维度拆解：不是在"整体好/不好"上达成一致，而是在具体维度上找到共识
3. 标准范例：建立公认的"好"和"不好"标杆样本，减少分歧
4. 保留多元标准：承认不同的审美取向，评测时从多个标准角度打分
```

### 2. "好"的标准在进化

模型的写作能力在提升，评测的标准也需要随之提升：

```
阶段 1：模型写的句子通顺 ← 评测标准：语法正确性
阶段 2：模型能写出完整段落 ← 评测标准：段落逻辑
阶段 3：模型能写出有结构的故事 ← 评测标准：情节设计
阶段 4：模型能写出有感染力的文字 ← 评测标准：情感表达
阶段 5：模型能写出有独特风格的作品 ← 评测标准：风格辨识度

随着模型能力的提升，评测的"及格线"也在不断上移。
```

### 3. 文艺写作评测的"自动化天花板"

与代码评测不同，创作领域的自动化评测有天然上限：

| 评测层次 | 自动化方法 | 可靠性 | 适用文体 |
|---------|-----------|-------|---------|
| **基础语法** | 自动语法检查 | ★★★★★ | 全部 |
| **格式规范** | 规则匹配 | ★★★★ | 公文/论文 |
| **语言流畅度** | PPL/Perplexity | ★★★ | 全部 |
| **结构完整性** | 规则+LLM-as-Judge | ★★★ | 报告/方案 |
| **逻辑一致性** | LLM-as-Judge | ★★ | 小说/论文 |
| **审美价值** | 必须人工 | ★ | 小说/诗歌 |

> 创作领域的自动化评测只能覆盖基础层面，**审美判断的核心仍然需要人类专家**——这是评测三部曲中最依赖人工的领域。

## 工程哲学

### 1. "AI 的壁垒，不过是尚未抵达的前沿"

这是整个知识库中最具哲学深度的工程信念：

> 审美不是 AI 的先天缺陷，而是一个尚未被攻克的工程挑战。写作与审美是人类认知中最抽象、最深邃的疆域，但这不等于 AI 不能到达——只是还没有人铺好路。

### 2. 审美标准是"训练数据"的一部分

在 AI 创作数据工程中，审美标准不是额外的"品味问题"，而是**数据集的核心组成部分**：

- 模型输出的质量上限 = 训练数据中"好文本"的最高水准
- 模型的审美判断力 = 评测体系中审美维度的精细化程度
- 因此，定义审美标准不是"锦上添花"，而是**决定模型创作能力上限的核心工作**

### 3. "功能写作"是创作能力的基础盘

虽然文艺写作听起来更"酷"，但功能写作可能是影响面更广的方向：

- 每个使用 Agent 的用户都需要"能用的文本"
- 搜索摘要、Agent 回答、报告生成……这些场景的文本质量直接影响用户体验
- 功能写作的评测标准相对明确，更适合作为创作的"起步区"

### 4. 三部曲：从实用到审美

三部曲的完整逻辑：

```
JD 11（通用）：解决"能不能用"的问题      ← 基础
JD 12（专业）：解决"对不对"的问题        ← 进阶
本 JD（创作）：解决"好不好/美不美"的问题  ← 最高
```

**每一层都建立在前一层的基础上**——先让模型能完成任务（JD 11），再让模型能给出专业正确的回答（JD 12），最后让模型能创作出有审美价值的文本（本 JD）。

## 知识冲突

### AI 能否真正理解"美"？
- **怀疑派**：审美是人类独有的情感体验，AI 只是模仿形式，不能真正理解美
- **乐观派**："美"可以被建模为形式特征和读者反应的映射关系，AI 可以学习并生成
- **调和**：AI 可能以不同于人类的方式"理解"美，但结果可以产生打动人心的作品——就像不同文化背景的人都能欣赏不同风格的文学

### 评测的自动化应该做到什么程度？
- **自动化派**：用 LLM-as-Judge 逐步替代人工评测，只有这样才能规模化
- **人工派**：创作评测必须由人类专家完成，LLM-as-Judge 会丢失最关键的审美信号
- **调和**：基础维度（语法/结构/格式）自动化，高级维度（审美/创意/情感）人工+LLM 辅助

### 两个方向是否需要不同的人？
- **合并派**：文本判断力是通用的，一个好的编辑能同时判断小说和报告的质量
- **分拆派**：文艺写作需要创作敏感性，功能写作需要逻辑和规范性——能力要求差异大
- **调和**：JD 放在同一职位下，说明公司认为基础审美能力可以迁移——但从长远看，两个方向可能会分化成不同的专业角色
