# DeepSeek 五份 JD 全景对比分析

> 基于 DeepSeek 已公开的五份招聘 JD（Harness 团队、服务端工程、预训练数据、AI 搜索、Agent Infra），从团队定位、技术栈、工程模型归属、产品形态等多维度进行全方位对比。
>
> 这五份 JD 共同构成了 DeepSeek 公司在大模型产业链中的**完整全局布局**。

## 二、核心维度对比矩阵

| 维度 | Harness 团队 | Agent Infra (DSec) | 服务端工程 | 预训练数据 | AI 搜索 |
|------|-------------|-------------------|-----------|-----------|---------|
| **领域** | Agent Runtime | 沙箱/虚拟化/基础设施 | 在线服务/基建 | 数据处理 | 信息检索 |
| **Pipeline 方向** | Agent Loop | 沙箱生命周期 | API→服务→数据 | 互联网→语料 | 互联网→检索 |
| **核心产出** | Agent 运行时设计/原则 | 沙箱平台可用性/安全性 | 在线服务可靠性 | 数据质量/规模 | 搜索质量/召回率 |
| **典型规模** | 百万级 Agent 任务 | 成千上万沙箱 | 千万日活 | TB 级 tokens | 千亿文档/千万日活 |
| **核心瓶颈** | Agent Loop 稳定性 | 隔离强度+资源效率 | 高并发低延迟 | 清洗质量与吞吐 | 召回精度与延迟 |
| **主力语言** | Rust（为主） | Rust/C/Python | C++/Python/Go | Python/C++/Rust | C++/Rust（架构）/Python（算法）|
| **三层模型重心** | Harness Engineering | 基础设施层 | Harness+基础设施 | Context Engineering | Context Engineering |
| **SOTA 目标** | Agent 能力的边界 | 沙箱安全/性能的 SOTA | 系统性能和可靠性 | 数据质量与多样性 | 召回率和搜索质量 |
| **核心对象** | Agent 执行控制 | 沙箱环境本身 | 系统可用性与效率 | 数据纯净度与规模 | 检索结果质量 |
| **用户类型** | 内部研究员/工程师 | Agent（非人类） | 外部亿万用户 | 内部模型训练团队 | 外部搜索用户+LLM |

> 🔑 **关键发现**：Agent Infra 的"用户"是 **Agent 本身**（非人类），这在五份 JD 中独一无二。

五份 JD 不是孤立的，"模型"是核心纽带：

1. **预训练数据团队** → 保障模型的"知识来源"（喂什么）
2. **AI 搜索团队** → 保障模型能"实时检索外部信息"（查什么）
3. **Harness 团队** → 将模型能力"封装为 Agent 产品"（怎么用）
4. **Agent Infra 团队** → 为 Agent 提供"安全运行场所"（在哪跑）
5. **服务端工程团队** → 为以上所有提供"在线服务底座"（怎么交付）

```
方向                   Python     C++      Rust     其他关键
─────────────────────  ───────  ───────  ───────  ──────────────

预训练数据                 ★★★★★    ★★★     ★★       SQL/Shell
AI 搜索（算法）            ★★★★★    ★★★     ★★       SQL
AI 搜索（架构）            ★★★     ★★★★★    ★★★★★    SQL/CUDA
```

### Rust 的全面崛起

五份 JD 中，**Harness 团队**、**Agent Infra**、**AI 搜索架构**、**服务端工程** 四份都将 Rust 列为核心或主力语言。Rust 已成为 DeepSeek **基础设施的默认语言**：

| Rust 的应用场景 | 对应 JD | 角色 |
|---------------|---------|------|
| Agent Runtime 核心 | Harness 团队 | **主力** |
| 虚拟化/沙箱基础设施 | Agent Infra (DSec) | **主力** |
| 搜索引擎核心 | AI 搜索架构 | **主力** |
| 在线服务/性能关键路径 | 服务端工程 | **主力之一** |
| 数据基建（KV/消息队列） | 预训练数据基建方向 | 加分/可选 |

> **结论**：DeepSeek 已成为 Rust 在 LLM 基础设施领域最大的企业级实践者之一。

## 六、从 JD 看 DeepSeek 的工程文化（更新版）

### 1. "节制"的工程哲学

五份 JD 中反复出现的关键词：

| 关键词 | 出现位置 | 含义 |
|--------|---------|------|
| **极致简洁 & 轻量** | AI 搜索 JD | 少依赖、低开销、快速迭代 |
| **不做过度设计** | AI 搜索算法方向 | 在成本与延时约束下落地算法 |
| **不过度搭建** | Harness（Bitter Lesson） | 能薄就薄，不堆 Scaffold |
| **让工程即作品** | 服务端工程 JD | 对代码质量的品味追求 |
| **硬功夫** | 预训练数据 JD | 基础工程能力的重要性 |
| **对简洁明了、可维护代码的基本美学素养** | Agent Infra JD | 写代码的"审美"是基本要求 |
| **另辟蹊径** | Agent Infra JD | 旧假设不成立时，不走老路 |

### 2. 全产业链布局（更新版）

```
研发层面：模型训练 ← 预训练数据（燃料）+ AI 搜索（感知）
产品层面：Agent 产品 ← Harness 团队（应用封装）
运行层面：Agent 沙箱 ← Agent Infra（物理载体）
交付层面：亿级服务 ← 服务端工程（在线底座）
```

DeepSeek 是全球少数同时拥有这五支团队的 AI 公司——从数据到搜索到 Agent Runtime 到沙箱环境到在线交付，**全栈自建**。

### 3. Rust 优先，系统能力为王

- 核心系统（Agent Runtime、沙箱、搜索引擎、在线服务）：**Rust 或 C++**
- 算法/数据处理/实验：**Python**
- Agent Infra 对 Rust/C 的要求最重——因为虚拟化/容器/存储/网络都是系统级编程的主力战场

### 4. 跨团队协作密度高

```
Harness 团队     ↔  研究员、产品经理、工程师
Agent Infra      ↔  模型训练团队、各 Agent 方向团队（DSec 直接服务于训练）
服务端工程        ↔  研究员（对内）、亿万用户（对外）
预训练数据        ↔  算法、训练、搜索、基座团队
AI 搜索           ↔  架构、数据、产品、posttrain 团队
```

Agent Infra 的"用户"是其他团队（尤其是模型训练团队）——这使它的协作模式更接近内部 PaaS 团队。

### 5. "高强度用户"门槛的延伸

DeepSeek 各 JD 中的"品味"要求链：

- **Harness** → "把 Agent 产品融入日常"（Agent 使用品味）
- **预训练数据** → "对模型行为有自己独特的理解"（数据品味）
- **AI 搜索架构** → "良好的系统设计品味"（系统品味）
- **Agent Infra** → "对简洁明了、可维护代码的基本美学素养"（代码品味）

> 品味（Taste）成为 DeepSeek 跨 JD 的**统一文化信号**——他们找的不只是能干活的人，而是有工程审美的人。

## 七、关键发现：Agent Infra 的独特定位

### 1. DSec 是 Harness 的"物理实现层"

```
Harness Engineering（抽象）：
Safety Layer → Sandboxed Execution → Retry/Recovery → Observability
        ↓ 物理实现
DSec（具体）：
VM/Container 隔离 → 临时存储/网络 → 节点自愈 → 监控告警
```

### 2. Agent 是"用户"——非人类用户

DSec 是五份 JD 中唯一以 **Agent（非人类）** 为主要用户的团队。这意味着：
- 不需要 UI/UX
- 需要有 API-first、Agent-first 的设计
- SLA/SLO 以 Agent 的任务成功率而非人类满意度衡量

### 3. 安全问题的维度升级

Agent Infra 面对的安全挑战比传统云平台多一个维度：

```
传统云安全：防外部攻击 → 防内部误操作
Agent 沙箱安全：防外部攻击 → 防 Agent 逃逸 → 防 Agent 间干扰 → 防 Agent 无意识自毁
```

### 4. 学术信号最强

Agent Infra JD 是五份中唯一明确列出**系统顶会（OSDI/SOSP/EuroSys/NSDI/ATC）**的——说明 DeepSeek 愿意在这个方向招聘纯系统背景的 PhD，而不要求 ML/AI 经验。
